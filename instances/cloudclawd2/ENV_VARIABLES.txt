## ============================================
## 📋 部署新服务时的修改清单
## ============================================
# 1. 修改实例名称（如 cloudclawd3）
# 2. 生成新的唯一 Token: openssl rand -hex 32
# 3. 填写新的飞书/钉钉凭证（支持多个同类型通道）
# 4. 确保不超过 8 个通道总数（所有通道类型之和）
# 5. 确保 OPENCLAW_WORKSPACE_DIR=/tmp/workspace
# 6. 确保 OPENCLAW_CONFIG_PATH=/data/openclaw/openclaw.json

## ============================================
## 📊 通道容量管理
## ============================================
# 本服务最多支持 8 个通道总数
# 容量计算示例：
# - 飞书 3 个 + 钉钉 2 个 + 企业微信 2 个 = 7 个 ✅ 可用
# - 飞书 5 个 + 钉钉 3 个 = 8 个 ✅ 可用
# - 飞书 5 个 + 钉钉 4 个 = 9 个 ❌ 超限
#
# 当某个服务满员时，新的通道申请将分配到其他有空位的服务
# 或创建新的服务实例（cloudclawd3, cloudclawd4, 等）
